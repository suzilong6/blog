# 关于this指向

this指向是JS中一个非常基础却又非常重要的知识点，接下来让我们学习一下this指向的四种情况。

# 默认绑定

即没有其他绑定规则存在时的默认规则。这也是函数调用中最常用的规则。

var name = 'suzilong';
function fn(){
    console.log(this.name); // suzilong
}
fn(); // fn() 等于 window.fn();

# 隐式绑定

函数的调用是在某个对象上触发的，即调用位置上存在上下文对象。

function say() {
    console.log(this.name);
}
const user = {
    name:'suzilong',
    action: {
        name:'action',
        say
    }
}
user.action.say(); // action
// 简单来说就是：谁调用say方法，say方法的this就指向谁
// 不管say方法前面有多少对象，我们只需要关注say方法前面那一个对象
// example: obj1.obj2.obj3.obj4.obj5.say(); say方法的this指向前一个对象obj5。

隐式丢失：

就是指隐式绑定的函数丢失绑定对象，从而默认绑定到全局或者undefined（取决于是否使用严格模式）

  1. 为函数调用创建别名
var name = 'window';
var obj = {
  name: 'suzilong',
  say() {
    console.log(this.name);
  }
}
const say = obj.say;
// 我们只需要关注是谁调用了say方法，很明显在全局作用域中是window调用了say，所以say中的this指向了window。
say(); // window

# 显式绑定（强制绑定）

通过使用 call, apply, bind 方法，强制绑定this。

var name = 'window';
function say() {
    console.log(this.name);
}
var user = {
    name:'suzilong',
}
say(); // window
say.call(user); // suzilong
say.apply(user); // suzilong
var bindSay = say.bind(user);
bindSay(); // suzilong

# new绑定

我们使用new来调用函数的时候，就会将构造函数中的this指向new出来的实例。

function User(){
    this.name = 'suzilong';
}
var user = new User();
console.log(user.name); // suzilong

# 绑定优先级

现在我们知道this绑定有四种情况，那么他的优先级是什么呢？

new绑定 > 显式绑定 > 隐式绑定 > 默认绑定

# 箭头函数的this

箭头韩式是ES6新增的语法，他的this绑定非常特殊：

箭头函数无法使用上述四条规则，而是根据外层（函数或者全局）作用域（词法作用域）来决定this。

简单来说就是：箭头函数不会绑定this，而是继承父作用域的this。

var name = 'window'
var user = {
    name: 'suzilong',
    say() {
        console.log(this.name)
    },
    sayName: () => {
        console.log(this.name)
    }
}
user.say() // suzilong
user.sayName() // window
// 箭头函数sayName的this绑定了它的父作用域window
