# 沙箱

沙箱英文 sandbox，简单来说是一种安全机制，为运行中的程序提供隔离环境。通常用于执行未经测试或不受信任的程序或代码，它会为待执行的程序创建一个独立的执行环境，内部程序的执行不会影响到外部程序的运行。

比如说 浏览器的 页签 也算一种沙箱，因为每一个 页签 都是独立的，不会相互影响，即使一个页签报错，崩溃，也不会影响其他页签的正常运行。

前端中也有很多地方会用到沙箱:

  • 在线代码编辑器，为了防止程序影响主页面，通常会把程序放在沙箱中执行，例如 codesandbox (opens new window)，码上掘金等
  • jsonp 请求的数据，第三方 js 的执行。如果未知其安全性，可以创建沙箱来解析数据/执行脚本
  • vue 服务端渲染、vue 模板中表达式计算等

# 快照沙箱

从名字上我们可以理解快照就是给你着一张相片，来记录你此刻的状态。qiankun的快照沙箱是基于diff来实现的，主要用于不支持window.Proxy的低版本浏览器，而且也只适应单个的子应用。

// src/sandbox/snapshotSandbox.ts
import type { SandBox } from '../interfaces';
import { SandBoxType } from '../interfaces';

/**
 * 迭代器方法
 * @param obj 
 * @param callbackFn 
 */
function iter(obj: typeof window, callbackFn: (prop: any) => void) {
  // eslint-disable-next-line guard-for-in, no-restricted-syntax
  for (const prop in obj) {
    // patch for clearInterval for compatible reason, see #1490
    if (obj.hasOwnProperty(prop) || prop === 'clearInterval') {
      callbackFn(prop);
    }
  }
}

/**
 * 基于 diff 方式实现的沙箱，用于不支持 Proxy 的低版本浏览器
 */
export default class SnapshotSandbox implements SandBox {
  proxy: WindowProxy; // window代理对象

  name: string; // 沙箱名称

  type: SandBoxType; // 沙箱类型

  sandboxRunning = true; // 当前沙箱是否正在运行

  private windowSnapshot!: Window; // 快照对象

  private modifyPropsMap: Record<any, any> = {}; // 在快照沙箱激活期间, 在 window 对象上修改的 属性/值

  constructor(name: string) {
    this.name = name;
    this.proxy = window;
    this.type = SandBoxType.Snapshot;
  }

  // 激活沙箱
  active() {
    // 记录当前快照
    this.windowSnapshot = {} as Window;
    // 将 window 浅拷贝给快照
    iter(window, (prop) => {
      this.windowSnapshot[prop] = window[prop];
    });

    // 恢复之前的变更
    Object.keys(this.modifyPropsMap).forEach((p: any) => {
      window[p] = this.modifyPropsMap[p];
    });

    this.sandboxRunning = true;
  }

  // 关闭沙箱
  inactive() {
    this.modifyPropsMap = {};

    // diff: 将 window 与 快照对象进行对比, 如果发现有不相同的就记录在 modifyPropsMap 中
    iter(window, (prop) => {
      if (window[prop] !== this.windowSnapshot[prop]) {
        // 记录变更
        this.modifyPropsMap[prop] = window[prop];
        // 恢复环境
        window[prop] = this.windowSnapshot[prop];
      }
    });
    
    this.sandboxRunning = false;
  }
}

# 支持单应用的代理沙箱

实现原理：通过 proxy 拦截对 window 的操作（get,set等），将这些操作的key和value保存下来，当沙箱开启时，会将上次沙箱开启时的操作记录复原到 window 上。当沙箱关闭时，会将 window 对象复原。

对比快照沙箱的优势是 不用 diff 去遍历整个 window 对象，性能有所提升。

class LegacySandBox {
  // 在window上新增的属性存在这里，比如声明一个全局变量 window.a = 1;
  addedPropsMapInSandbox = new Map();
  // 在window上修改原生的属性存在这里，比如我重写了console: window.console = xxx。
  // 那么modifiedPropsOriginalValueMapInSandbox 会报错 console 和 console 的原始值
  modifiedPropsOriginalValueMapInSandbox = new Map();
  // 在window上修改的属性都存在这里，不管是新增的还是更新
  currentUpdatedPropsValueMap = new Map();
  proxyWindow;
  // 设置window的属性/值
  setWindowProp(prop, value, toDelete = false) {
    if (value === undefined && toDelete) {
      delete window[prop];
    } else {
      window[prop] = value;
    }
  }
  // 激化沙箱
  active() {
    // 将上一次激活沙箱期间的改动复原
    this.currentUpdatedPropsValueMap.forEach((value, prop) => this.setWindowProp(prop, value));
  }
  // 关闭沙箱，将本次沙箱激活期间对window所作的操作复原
  inactive() {
    this.modifiedPropsOriginalValueMapInSandbox.forEach((value, prop) => this.setWindowProp(prop, value));
    this.addedPropsMapInSandbox.forEach((_, prop) => this.setWindowProp(prop, undefined, true));
  }
  constructor() {
    const fakeWindow = Object.create(null);
    // 设置代理，关键
    this.proxyWindow = new Proxy(fakeWindow, {
      set: (target, prop, value, receiver) => {
        const originalVal = window[prop];
        // window 上没有的属性,属于新增,加到 addedPropsMapInSandbox 里
        if (!window.hasOwnProperty(prop)) {
          this.addedPropsMapInSandbox.set(prop, value);
        } else if (!this.modifiedPropsOriginalValueMapInSandbox.has(prop)) {
          // window 上有的属性, modifiedPropsOriginalValueMapInSandbox 里没有
          this.modifiedPropsOriginalValueMapInSandbox.set(prop, originalVal);
        }
        // 记录所有修改的属性
        this.currentUpdatedPropsValueMap.set(prop, value);
        window[prop] = value;
      },
      get: (target, prop, receiver) => {
        return target[prop];
      },
    });
  }
}

# 支持多应用的代理沙箱

ProxySandBox 的实现原始是 通过拦截对 window 对象的操作（get,set等），对 window 对象的操作都执行在代理对象上。相对于快照沙箱和单应用代理沙箱，优势在于不会在 window 对象上直接操作，不会污染 window 对象，因此也可以支持多应用。

class ProxySandBox{
    proxy;
    isRunning = false;
    active(){
        this.isRunning = true;
    }
    inactive(){
        this.isRunning = false;
    }
    constructor(){
      // 对 window 的所有操作 （get,set）, 都代理到 fakeWindow 上，不再操作 window 对象
        const fakeWindow = Object.create(null);
        this.proxy = new Proxy(fakeWindow,{
            set:(target, prop, value, receiver)=>{
                if(this.isRunning){
                    target[prop] = value;
                }
            },
            get:(target, prop, receiver)=>{
              if (this.isRunning) {
                // 先在 fakeWindow 上找，到不到再去 window 上找
                return  prop in target ? target[prop] : window[prop];
              } else {
                return  window[prop];
              }
            }
        });
    }
}

const sandbox1 = new ProxySandBox();
const sandbox2 = new ProxySandBox();
((window) => {
   sandbox1.active();
   window.nickname = 'sandbox1';
   console.log(window.nickname); // sandbox1
  
   sandbox1.inactive();
   console.log(window.nickname); // undefined
  
   sandbox1.active();
   console.log(window.nickname); // sandbox1
})(sandbox1.proxy);
((window) => {
   sandbox2.active();
   window.nickname = 'sandbox2';
   console.log(window.nickname); // sandbox2
  
   sandbox2.inactive();
   console.log(window.nickname); // undefined
  
   sandbox2.active();
   console.log(window.nickname); // sandbox2
})(sandbox2.proxy);
